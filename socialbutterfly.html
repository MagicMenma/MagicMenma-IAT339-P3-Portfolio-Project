Social Butterfly

In the final project of IAT 320 Body Interface, we delve into the innovative use of the human body as an input device, extending beyond traditional interfaces. This virtual game, crafted for two players, not only serves as an engaging platform but also as a means to explore and enhance social connections. Our approach integrates the sophisticated use of OpenCV in Python for computer vision, allowing for an immersive interaction design through camera-based tracking.

OpenCV & Data Exchange

The main data acquisition device in the system is a webcam, which is connected to the host computer via a USB interface. The task of this webcam is to acquire the motion data of the two participants. The motion data of the participants is transferred to the Python environment as soon as it is captured. We use the OpenCV library here to process the video stream.

After motion analysis, the system identifies and records key location points on the participant's body, with each participant's data consisting of 32 different points. I then combine and serialize this data and apply a smoothing algorithm on two frames to improve the motion quality. The serialized data string is transferred to the Unity software through a server address port.

Problem Solving & C# Coding

Once I had gathered all the necessary reference materials, I began the modeling process. This involved creating a basic 3D model in Blender, which I could then refine and add detail to over time.

As part of the modeling process, I also had to ensure that the model was properly UV mapped. This involves editing the UVs of the model so that materials can be applied correctly.UV editing can be a time-consuming process, but it's essential for creating high-quality 3D models. It allows for greater control over how textures are applied to the model, and can help to ensure that the final product looks as realistic as possible.

Enviroment & Levels Setting

After completing the 3D model in Blender, I imported it into Twinmotion to further enhance the project. Twinmotion is a powerful real-time rendering software that adds textures, materials, and other visual elements to the 3D model.

One of the first things I did in Twinmotion was to add texture maps to the model. This involved applying materials to various surfaces of the model, such as the walls of buildings, pavement, and grass. I also added plant and vegetation materials to the model to make it look more realistic and natural. Another important feature was the ability to add roads and driveways to the model. I created a detailed network of roads around the park to make the model more like a realistic environment.

Reflection

Attention to detail is crucial: Creating a highly detailed and accurate 3D model requires a lot of attention to detail. From gathering reference materials to optimizing the model for use in Twinmotion, every step of the process required a high level of precision and care.

Technical skills are important: The project required a range of technical skills, including proficiency in Blender and Twinmotion. While these skills can be challenging to learn, they are essential for creating high-quality 3D models.

Overall, I found the experience of creating a 3D model for Holland Park to be a valuable learning opportunity. The project challenged me to develop new technical skills. The combination of Blender and Twinmotion allowed me to create a highly detailed and visually stunning 3D model of Holland Park that would serve as an excellent showcase for student work.
